# Cookbook style guide

## Purpose
This cookbook is practical first: recipes should be easy to cook from directly, with strong authenticity and consistent formatting.

## User taste and house rules

### Indian flavour preferences
1. North Indian homemade dishes: prefer UP Kayastha style.
2. South Indian dishes: prefer Bangalore flavour.
3. Indo-Chinese dishes: prefer North Eastern hills flavour.
4. Regional Indian dishes: prefer regional accuracy over generic "restaurant style".

### General preferences
- Authenticity matters most.
- Home-style recipes are preferred unless explicitly asking for restaurant or dhaba style.
- Avoid unnecessary cream, sugar, or tomato overload.
- Keep recipes realistic for a UK kitchen.
- Prefer concise instructions with useful cues.
- Use 2-space indentation when indentation is needed in code or structured examples.

## Writing style
- Direct and calm.
- No fluff.
- No fake storytelling.
- No marketing tone.
- No excessive headers.

## Units
- Metric first.
- Spoon measures are acceptable where natural.
- Use weights for major ingredients where helpful.

## Recipe structure
Use this order:
1. Summary
2. Metadata
3. Ingredients
4. Method
5. Notes
6. Serving suggestions
7. Personal notes
8. Change log (optional)

## Ingredient language
- Use the ingredient name the user is likely to buy in the UK.
- Add Indian names in parentheses only when genuinely helpful.
- Avoid unexplained US-only terms.

## Authenticity markers
Where relevant, identify the style:
- homestyle
- dhaba-style
- darshini-style
- Kolkata street-style
- Delhi restaurant-style
- Bangalore home-style
- etc.

## Variants
Variants should be clearly labelled:
- quick version
- guest version
- lighter version
- no-onion/no-garlic
- oven version
- pressure cooker version
- slow cooker version

## Personal notes
If the user has actually cooked the dish, preserve:
- what worked
- what failed
- texture outcomes
- timing tweaks
- equipment-specific notes
- family corrections

Never overwrite these.
